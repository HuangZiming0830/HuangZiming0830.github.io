Gradient Boosting Descent Machine

  • GBDM: 每次迭代都需要遍历整个训练数据多次

  • XGBOOST: 基于预排序方法的决策树算法

    • Level-wise 增长策略:不容易过拟合，容易进行多线程优化，控制模型复杂度，但效率较低
    • Modified Preorder Tree Taversal: 用于层级关系的存储和遍历, 解决邻接表递归查询低效问题，但不适用于增删改操作较多的场景
            # 创建数据表
      CREATE TABLE 'product_category' (
          'id' int(11) NOT NULL AUTO_INCREMENT,
          category_name varchar(20) not null,
          left_value int(10) not null,
          right_value int(10) not null,
          PRIMARY KEY ('id')) ENGINE=InnoDB DEFAULT CHARSET=utf8
            # 插入子分类
      INSERT into product_category(category_name, 'left_value', 'right_value')
      # 插入数据
      values('food', 1, 2);
      # 锁表 防止被同时修改导致数据不一致
      lock table 'product_category' write;
      # 插入子分类
      select @myLeft := 'left_value' 
      from 'product_category'
      where category_name = 'food';
      # 更新受影响分类左右区间值
      update product_category SET left_value=left_value +2 where left_value>@myLeft;
      update product_category
      SET right_value=right_value + 2 where right_value>@myLeft;
      # 插入子分类信息
      INSERT into product_category(category_name, 'left_value', 'right_value')
      values('meat', @myLeft+1, @myLeft+2);
      # 解除锁表
      unlock tables;
            # 查询指定分类子分类
      select @myLeft := 'left_value', @myRight := 'right_value'
      from 'product_category'
      where 'category_name'='meat';
      select * 
      from 'product_caegory'
      where 'left_value' >= @myLeft and 'right_value' <= @myRight;

  • LightGBM:

    • 基于Histogram的决策树算法
      • 内存占用少，计算代价小
      • 差加速: 叶子节点直方图由父节点和相邻节点作差得到，只使用非零特征构建直方图
    • Leaf_wise 增长策略
      • 在分裂次数相同的情况下，可以降低更多的误差
      • 但可能会长出比较深的决策树，产生过拟合，需要增加最大深度限制，在保证高效率的同时防止过拟合
    • Gradient-based One-Side Sampling: 从减少样本的角度出发，排除大部分对计算信息增益没有帮助的小梯度样本
